Challenges

Keystone was newly developed, and while the atomic core layer was established first, a significant gap emerged between the new core library and the existing legacy system.

  1. Legacy-to-core disconnect
  2. Pattern inconsistency
  3. Migration bottleneck

My role

Led the full ecosystem for TikTok Ads Manager Library, and TikTok One Library.

  • Component strategy & roadmap
  • Cross-functional collaboration (designers and engineers)
  • Creation, maintenance, documentation, and governance
  • Migration support & deprecation system

How to solve?

→ Introducing a tier structure helps the management of the core and composite libraries

Core Library (Tier 1) is a high-quality, centrally governed set of atomic and foundational components used across all platforms. Platform Libraries (Tier 2) is composite components built from core primitives, tailored for the unique needs of each product platform.

Why tiers are divided by platforms, not features?

Each TikTok monetization product has distinct workflows and business logic.

  • TikTok Ads Manager (TTAM): Heavy use of forms for campaign creation, tables, and analytical displays.
  • TikTok One: Search, discovery, and lists of creators and partners for collaboration workflows.
  • Platform libraries allow product-specific structures while sharing the same foundational behavior, tokens, and interaction patterns.

→ Platform libraries provide composite components

Platform components are organism-level composite components, created by combining core components. They deliver reusable solutions and patterns for a consistent user experience across contexts.

Key rules: Never break the foundation chain

Platform components must inherit styling, interactions, and tokens from core components, providing scalable complexity without breaking foundations.

How platform components contribute to products

This example shows how platform components provide reusable patterns built on core components. This efficient support enables effective page migrations and solves legacy system gaps.

Strategy of the platform components

→ Use of slots

Platform extends with context: the platform component leverages the core tile to insert customized designs, creating specific platform tile components. In this way, the platform component maintains the tile's interaction states, styling, and foundations for both design and code sides.

→ Provide pattern consistency

Platform components deliver preset layout patterns backed by foundation tokens, ensuring interactions remain aligned with core standards. Frequently used platform components can transition into the core library, strengthening shared patterns and enhancing design efficiency.

How to evaluate component's tier

To streamline component creation and management, work with both the core and platform libraries to define clear evaluation criteria. Using a checklist approach, a requested component is assigned to the appropriate tier when it meets the required checkpoints for that category.

Collaboration working model

The collaboration model shows how the product design and design system teams work in tandem from kick-off to release. Product designers identify needs and request components, while the design system team evaluates these requests, understands product scenarios, and defines the supporting patterns. The two teams align at key checkpoints, such as reviews and adoption testing. After finalizing documentation and supporting engineering implementation through design QA, the design system team hands off the implemented components to product engineers. As the product design team proceeds with product design QA, the design system team refines documentation and publishes updates to the Figma library.

Business impact

  1. Developed 30 TikTok Ad manager platform components, and Achieved 27K Figma insertions, demonstrating strong adoption and measurable improvement in design quality in 2025.
  2. Consolidated 6 redundant local video tile designs into a single unified video tile, achieving an 83% reduction in duplicated design work. Leveraged a single core asset tile with custom slot variants to support 20 video-tile variants, eliminating the need to create 20 separate components. This approach drove a 95% reduction in variant duplication, strengthening design consistency, reducing waste, and significantly improving reusability across teams.

Learning

  2. Improving alignment and support between product launch timelines and platform component release schedules.
  3. Further thoughts on efficient solutions and defining what an ideal automated code migration for platform libraries should achieve.
